1) Working principles

Kolosso-eventer decorates a JS Object or Function providing first of all the two basic methods on(event_type, callback) and trigger(event_type).

var one = ke({});
var two = ke(function(){});
two.on('event_type_1', function(){ alert('got an event_type_1'); });

2) Listening to oneself

A kolosso-evented object or function can first of all listen to its own events.
This is mainly useful when a complex widget must synchronize its own parts.

var one = ke({});
one.on('event_type_2', function(){ alert('got an event_type_2'); });

3) Event payloads and multiple callbacks

Each trigger command can add a payload to its event.
Listeners can attach many callbacks to the same event type. There is no guarantee on the order callbacks are executed

var one = ke({});
var two = ke(function(){});
two.on('event_type_3', function(payload){ alert('got an event_type_3'); });
two.on('event_type_3', function(payload){ alert('payload=' + payload); });

4) Subscribers, publishers and global events

Pub-Sub schema's involving two or more objects/functions are commonplace.
The 

ke
object itself is a publisher, to whom any kolosso-evented instance may listen.

var two = ke(function(){});
